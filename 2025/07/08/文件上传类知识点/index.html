文件上传类知识点

简介

文件上传本质上就是将文件传输到服务器时没有做好检测和过滤，导致可上传恶意程序到服务器，从而获得后台权限来干点坏事。一般这个恶意脚本文件称为webshell（恶意脚本程序），通过webshell我们可以查看服务器的目录结构或文件，以及来执行一些系统指令。小马和大马，简单来说，大马是多功能，危害大，体积大的木马，小马是隐蔽性强，体量小，更方便的小木马。一句话木马就属于小马，所以较易突破防护，而大马极易被过滤。

主要使用工具

（1）一句话木马

通过文件上传将一句话木马上传到服务器，我们就可以进行一系列的操作来对这个服务器的数据库进行入侵，从而可以从数据库中得到我们想要的数据。

（2）蚁剑

上出啊一句话木马成功之后我们可以通过蚁剑来连接我们上传攻击的网站，从而可以得到他的有用信息。

题目类型

1.无验证类型

这种类型的题目最简单，没有任何的文件验证，我们可以直接上传一个一句话木马的文件来获取flag。

2.客户端验证（前端验证）

有些题目他在他的js前端会检测我们文件的类型，我们可以先上传他要求的文件类型，让后再在Burp里面将文件类型改回。在 Burp Suite 中修改上传请求可以绕过前端验证，主要是因为 前端验证仅依赖浏览器端的 JavaScript 检查，而 HTTP 请求在发送到服务器之前可以被拦截并篡改。意思就是burp在浏览器前端验证过之后发给服务器的过程上把这个信息给截断并且篡改了。

3.服务端验证（后端验证）

也就是后端检测，这些不可以直接在Burp改后缀，他们通常在php，java，node.js中运行检测代码。

后缀检测

1，黑名单检测

这类题一般都是使用黑名单，将我们要上传文件的一些后缀名给禁用了，我们可以通过一系列的操作来进行黑名单绕过。

（1）双写绕过（ctfhub）

这同题目就是服务器在检测到你的文件后缀为php的时候他会把这个后缀给删掉，从而让你的代码无法编译运行，这类题也很简单，我们自需要加上双写后缀就可以解决。如.pphphp。

例题

我们带题目上传一个php文件，显示上传成功，但是我们文件的php后缀没有了，就可以得知这可以试一下双写绕过。

我们改一下后缀，改成.pphphp.上传后发现后缀就成为php了，因此这个题目会将文件后最的第一个php给去掉，我么就可以用双写后缀来解决这个题目。

然后直接用一件链接在flag.php中遭到了本题的flag。

我么在发现php等后缀被完全禁止了的时候可以试一下phtml后缀，这个后缀同样也可以对php代码进行解析并运行。.phtml 后缀的文件可以解析 PHP 代码，主要是因为 Web 服务器（如 Apache/Nginx）的配置默认将其视为 PHP 可执行文件。

这列题目会检测我们文件的一些内容，从而达到限制文件上传的目的。
