SQL注入基础

一，数据库

SQL（Structured Query Language）是数据库专属语言，是用来和数据库对话的语言，用于管理关系数据库管理系统。

1.数据库基本结构

库 表 列 数据

2.关系型数据库

Oracle、MSSQL、MySQL、PostgreSQL、IBM DB2、Access等。

3.非关系型数据库（NoSQL数据库）

MongoDB、Redis、Memcached等

二，MySQL语法

1.关键词，函数，特性

1.version() # 查看数据库版本
2.database() # 查看当前数据库名
3.user() # 查看当前数据库用户
4.system_user() # 查看系统用户名
5.group_concat() # 把数据库中的某列数据或某几列数据合并为一个字符串
6.@@datadir # 查看数据库路径
7.@@version_compile_os # 查看操作系统

8.concat(),concat_ws(),group_concat() #字符串拼接

9.order by 按列排序，判断原始的select语句有几列

10.union select 联合查询，前后两次查询，字段数相同

  1. LIMIT N,M 用于限制查询结果返回行数，N表示从结果集的第N+1行开始返回默认0，M表示最多返回行数

12.group by 根据一个或多个列对结果集进行分组，可结合一些聚合函数来使用

13.where 条件语句 AND OR

14.隐式类型转换 - 数字、字符串、HEX()、ASCII()

15.MySQL 5.0版本以上，自带数据库information_schema包含数据库结构信息

16.表名和字段名可以通过反引号```使用关键字

2.注释语法

1.行间注释

  • -- 注意后面有空格

  • #

2.行内注释

  • /*注释内容*/

  • /*! 注释内容*/

3.文件操作

MySQL支持读写文件，但与配置有关

  • 文件的位置必须在服务器上，必须知道绝对路径，有file权限
  • 文件可读取，文件大小小于max_allow_packet字节
  • 如不满足条件，返回NULL

三，SQL注入概述

SQL注入是一种网络安全攻击技术（web安全漏洞），攻击者可以执行恶意SQL语句。利用SQL注入漏洞，攻击者可以检索、添加、修改和删除数据库中的记录，甚至可以获取数据库服务器权限。

1.产生条件

①参数是用户可控的，也就是前端传入后端的参数的内容是用户可以控制的；（用户控制输入）

②参数被带入数据库进行查询，也就是传入的参数被拼接到SQL语句中，并且被带入到数据库进行查询；（程序可以执行拼接了用户输入的SQL语句）

2.产生危害

1.数据库信息泄露：泄露数据库中存放的数据、用户隐私等

2.获取WebShell：当前用户为高权限（例如root）且知道绝对路径时，可以直接写一句话木马到服务器。

3.网页篡改：注入出后台管理员用户，登陆后台后发布恶意数据、篡改后台数据等。

4.网站挂马 : 当拿到WebShell或者获取到服务器的权限后，可将一些网页木马挂在服务器上，去攻击别人。

5.获取系统权限：当权限足够高时，可以获取系统主机的权限。

6.绕过登录验证：使用万能密码登录网站后台等

7.文件系统操作：列目录，读取，写入文件等

四，SQL注入流程

1.寻找注入点以及判断整数型还是字符型

注入点都是与数据库有交互的地方 ‘的url编码是%7

常用注入点：①表单中的输入参数②URL中的参数③Cookie或header头中其它参数

一般引入单引号判断是否存在注入点

2.判断闭合符号

常用测试符号：‘ “ ’） ”）

**注意：**是字符型才要猜闭合，如果是数字型可以跳过这个步骤。在字符型闭合后，在后面加上– +（中间有一个空格），可以将原本后面的代码注释掉。

输入’时报错说明是字符型无报错说明是数字型

根据报错信息判断闭合符号

输入常用闭合符号若报错，根据报错信息可以发现闭合符号

最外层单引号是mysql报错自己加的，去掉它自己加的以及我们自己输入的值，则判断出后台包裹变量字符串的符号为 单引号 ‘

使用注释符号注释掉后面的sql语句，sql能正常执行

所以注入点为：id,闭合符号：’（单引号）

3.查询列数

用order by语句判断列数

从1开始尝试3时正常4时报错，确认列数为3

五，SQL注入类型

1.联合查询注入（UNION query-based）

条件：有注入点且数据有回显

看上述

2.报错注入（error-based）

存在注入，且有错误信息显示，通过人为制造错误条件，使得结果出现在错误信息中。

报错注入是通过特殊函数错误使用并使其输出错误结果来获取信息的。

注：~按位取反

例子：less-5

①floor报错注入

floor()报错注入的原因是group by在向临时表插入数据时，由于rand()多次计算导致插入临时表时主键
重复，从而报错，又因为报错前concat()中的SQL语句或函数被执行，所以该语句报错且被抛出的主键
是SQL语句或函数执行后的结果

  • rand()函数:随机返回0-1的小数
  • floor()函数：小数向下取整，向上取整ceiling函数()
  • concat_wa(““,2,3)函数“将括号内数据用第一个字段连接起来:23
  • group by():分组查询
  • count():统计
  • limit():显示指定行数

②updatexml报错注入

③extractvalue报错注入

extractValue()函数在SQL语句中用于提取XML类型的数据中指定节点的值。

接受两个参数：第一个参数是XML类型的列或表达式，第二个参数是XPath表达式。XPath是一种用于在XML文档中定位节点的语言。

extractValue()函数首先对XML数据进行解析，然后基于XPath表达式定位到指定的节点，最后返回该节点的值。如果节点不存在，函数将返回空值。

3.盲注

存在注入，但没有回显和错误信息。盲注根据判断指标，分为基于布尔的盲注和基于时间的盲注。

  • SUBSTR(_string_, _start_, _lenth_) - 字符串截取
  • ASCII(_character_) - 返回字符的ASCII值
  • LENGTH(_string_) - 返回字符串长度
  • if(条件,成立,不成立)
  • SELECT IF(500<1000, "YES", "NO");

时间的盲注（time-based blind）

时间盲注的一般思路是延迟注入，就是利用sleep()或benchmark()等函数让mysql执行时间变长并结合判断条件语句if(expr1,expr2,expr3)，然后通过页面的响应时间长短来判断语句返回的值是True还是False，从而猜解一些未知的字段

相关函数

if(expr1,expr2,expr3)： expr1的值为TRUE，则返回值为expr2 ；expr1的值为FALSE，则返回值为expr3
sleep(n)：延迟响应时间n秒

布尔的盲注（boolean-based blind）

布尔盲注一般适用于页面没有回显字段不支持联合查询和报错注入，且web页面返回True 或者 false，构
造SQL语句，利用and，or等关键字来其后的语句 true 或 false 使web页面返回 true 或 false，从而
达到注入的目的来获取信息。

相关函数

（1）length：返回值为字符串的字节长度
（2）ascii：把字符转换成ascii码值的函数
（3）substr(str, pos, len)：在str中从pos开始的位置（起始位置为1），截取len个字符
（4）count：统计表中记录的一个函数，返回匹配条件的行数
（5）limit：
limit m ：检索前m行数据，显示1-10行数据（m>0）
limit(x,y)：检索从x+1行开始的y行数据

4.堆叠注入（Stacked Queries）

一次执行多条SQL语句，每条语句以;结尾。比如后端使用mysqli_multi_query函数。由于可以执行其他语句，堆叠注入的危害性更大。

5.二次（阶）注入（Double Order SQLi）

二次注入是指已存储（数据库、文件）的用户输入被读取后再次进入到 SQL 查询语句中导致的注入。

  • addslashes，仅仅是为了获取插入数据库的数据，额外的\并不会插入

六，sqlmap工具

1.确认注入点：sqlmap -u URL

2.获取库的信息：sqlmap -u URL –dbs（爆出所有库的信息）sqlmap -u URL –current-db（爆出当前库的信息）

3.获取表的信息：sqlmap -u URL -D maoshe（当前库名） –tables

4.获取列的信息：sqlmap -u URL -D maoshe（当前库名） -T admin（当前表名） –columns

5.获取字段数据：sqlmap -u URL -D maoshe（当前库名） -T admin（当前表名） -C Id,password,username –dump
